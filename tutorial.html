OVERVIEW

このチュートリアルで作るもの

Claude Codeと一緒に、業務で使える日報作成ツールをゼロから作る

このチュートリアルでは、Claude Codeを使って「日報作成ツール」を1から作り、GitHubにプッシュし、GitHub Pagesで公開するところまでを体験します。プログラミングの経験がほとんどなくても、手順通りに進めれば完成できるように構成しています。

完成する日報ツールの機能

📝
日報の入力
日付、今日やったこと、明日やること、困っていること、メモを入力できるフォーム
💾
保存と閲覧
入力した日報をブラウザに保存（localStorage）。過去の日報を一覧で確認できる
📋
コピー＆共有
日報をテキスト形式でクリップボードにコピー。SlackやTeamsに貼り付けて共有できる
💡 なぜ「日報ツール」なのか

日報ツールは、フォーム入力・データの保存・一覧表示・テキスト出力という、Webアプリの基本的な要素をすべて含んでいます。かつ、業務でそのまま使える実用性もあります。HTML・CSS・JavaScriptだけで動くため、サーバーが不要でGitHub Pagesでそのまま公開できる点もチュートリアルの題材として適しています。

STEP 0

始める前に — 必要なアカウントの準備

ツールのインストールに入る前に、2つのアカウントを用意する ⏱ 5–10分

Claude CodeとGitHubを使うために、以下の2つのアカウントが必要です。すでに持っている場合はStep 1に進んでください。

1. Anthropicアカウント（Claude用）

Claude Codeを使うには、Anthropicのアカウントと有料プラン（Pro以上）またはAPIクレジットが必要です。

  • claude.ai でアカウントを作成する メールアドレスまたはGoogleアカウントでサインアップ
  • Claude ProまたはMaxプランに加入する（月額$20〜） Free プランではClaude Codeは利用できない。あるいはAPI利用（従量課金）でも可
💡 プランの選び方

学習目的ならClaude Pro（月額$20）で十分です。APIの従量課金を選ぶ場合は、console.anthropic.com でAPIキーを取得し、クレジットをチャージしてください。このチュートリアルの日報ツール開発で使用するAPI費用は数百円程度です。

2. GitHubアカウント

コードの管理と公開に使います。無料プランで十分です。

  • github.com/signup でアカウントを作成する ユーザー名は公開URLの一部になる（例: username.github.io）。後から変更も可能
STEP 1

ツールのインストール

Node.js → Claude Code → VS Code → Git → gh CLI の順に入れていく ⏱ 15–30分

開発に必要なツールを順番にインストールします。各ツールが何のために必要かも簡単に説明します。

1-1. Node.js のインストール

Node.js（ノードジェイエス）は、JavaScriptをパソコン上で実行するための環境です。Claude Code自体がNode.jsで動いているため、先にインストールが必要です。

nodejs.org にアクセスし、LTS版（長期サポート版）をダウンロードしてインストールしてください。

インストールが完了したら、ターミナルを開いて以下のコマンドで確認します。

💡 ターミナルの開き方

Mac: Spotlight検索（⌘+Space）で「ターミナル」と入力して開く。
Windows: スタートメニューから「PowerShell」（Windowsのターミナル）を検索して開く。

$ node --version
v22.x.x（数字は異なっていてOK。v18以上であれば問題なし）
command not found: node と表示された場合、ターミナルを一度閉じて開き直してください。それでもダメな場合は、Node.jsのインストーラをもう一度実行してみてください。
Node.jsのインストーラは Windows / Mac 共通で nodejs.org からダウンロードできます。Windowsでは .msi 形式のインストーラを使ってください。インストール後はPowerShellを管理者として実行して確認してください。

1-2. Claude Code のインストール

Claude Codeはターミナルで動くAIコーディングツールです。npmという（Node.jsに付属する）パッケージ管理ツールでインストールします。

$ npm install -g @anthropic-ai/claude-code

インストールが終わったら、確認します：

$ claude --version
バージョン番号が表示されればOKです。

次に、Claude Codeを起動してアカウントを接続します：

$ claude

初回起動時にブラウザが開き、Anthropicアカウントでのログインを求められます。画面の指示に従ってログインしてください。完了するとターミナルに戻り、Claude Codeが使えるようになります。

確認が終わったら、いったん /exit と入力してClaude Codeを終了しておきます。

> /exit
npm: permission denied と出た場合（Mac）：
sudo npm install -g @anthropic-ai/claude-code で再実行してください（パスワード入力を求められます）。
Windowsでは、PowerShellを管理者として実行してからインストールコマンドを実行してください。右クリック →「管理者として実行」で開けます。

1-3. VS Code のインストール

VS Code（Visual Studio Code）は無料のコードエディタです。AIが書いたコードを「見る」ための道具として使います。

code.visualstudio.com からダウンロードしてインストールしてください。

インストール後、ターミナルから code コマンドで起動できるようにしておくと便利です：

💡 Mac: code コマンドの設定

VS Codeを開き、⌘+Shift+P でコマンドパレットを開く → 「Shell Command: Install 'code' command in PATH」を選択。これで、ターミナルから code . と打つとVS Codeが開くようになります。

Windowsではインストール時に「PATHに追加」にチェックを入れてください（デフォルトで有効のはず）。インストール後にPowerShellで code --version を実行して確認できます。

1-4. Git のインストールと初期設定

Git（ギット）はコードのバージョン管理ツールです。「いつでも前の状態に戻せる」安全装置として機能します。

Macの場合、Gitはすでに入っていることが多いです。確認してみましょう：

$ git --version
git version 2.x.x と表示されれば、すでにインストール済みです。

入っていない場合、Macでは以下を実行するとインストールが始まります：

$ xcode-select --install
git-scm.com からWindows用インストーラをダウンロードしてください。インストール時の設定は基本的にデフォルトでOKです。インストール後、PowerShellを開き直してから git --version で確認してください。

Gitの初期設定として、ユーザー名とメールアドレスを登録します。これはコミット（変更記録）に表示される情報です：

$ git config --global user.name "あなたの名前"
$ git config --global user.email "あなたのメールアドレス"
💡 GitHubと同じメールアドレスにする

ここで設定するメールアドレスは、GitHubアカウントに登録したものと同じにしてください。GitHubがコミットとアカウントを正しく紐づけるために必要です。

1-5. gh CLI のインストール

gh CLI（GitHub CLI）は、ターミナルからGitHubの操作ができるツールです。リポジトリの作成やPagesの設定をコマンドラインから行えます。

Mac（Homebrewが入っている場合）：

$ brew install gh
💡 Homebrewが入っていない場合

Homebrew（ホームブリュー）はMac用のパッケージ管理ツールです。未インストールの場合は brew.sh の手順に従ってまずHomebrewを入れてから brew install gh を実行してください。Homebrewなしの場合は cli.github.com からインストーラをダウンロードすることもできます。

winget install --id GitHub.cli で入ります。あるいは cli.github.com からインストーラをダウンロードしてください。

インストール後、GitHubアカウントでログインします：

$ gh auth login

対話形式でいくつか質問されます。以下のように選択してください：

質問選択
Where do you use GitHub?GitHub.com
What is your preferred protocol for Git operations?HTTPS
Authenticate Git with your GitHub credentials?Yes
How would you like to authenticate GitHub CLI?Login with a web browser

ブラウザが開いてGitHubへのログインを求められます。承認すると、ターミナルに戻って完了です。

インストール完了の確認

すべてのツールが正しくインストールされたか、まとめて確認しましょう：

$ node --version && npm --version && git --version && gh --version && claude --version
5つのバージョン情報がすべて表示されれば、インストールは完了です。エラーが出たツールがあれば、そのツールのインストール手順を見直してください。
STEP 2

プロジェクトを作成する

作業フォルダを作り、GitリポジトリとGitHubリポジトリを準備する ⏱ 5–10分

2-1. プロジェクトフォルダの作成

ターミナルで、日報ツールのプロジェクトフォルダを作ります。

$ mkdir daily-report-tool && cd daily-report-tool

mkdir はフォルダを作るコマンド、cd はそのフォルダに移動するコマンドです。

コマンドはMacと同じです。PowerShellで実行してください。

2-2. Gitリポジトリの初期化

このフォルダをGitで管理できるようにします。

$ git init
Initialized empty Git repository in /Users/.../daily-report-tool/.git/

2-3. GitHubにリモートリポジトリを作成

gh CLI を使って、GitHubにリポジトリ（プロジェクトのフォルダ）を作成し、ローカルと接続します。

$ gh repo create daily-report-tool --public --source=. --remote=origin

このコマンドの意味：

部分意味
gh repo createGitHubにリポジトリを作成する
daily-report-toolリポジトリの名前
--public公開リポジトリにする（GitHub Pagesに必要）
--source=.現在のフォルダをソースとして使う
--remote=origin「origin」という名前でGitHubとの接続を設定
✓ Created repository あなたのユーザー名/daily-report-tool on GitHub
✓ Added remote https://github.com/あなたのユーザー名/daily-report-tool.git

2-4. 最初のコミットを作る

Gitの基本的なルールとして、空のリポジトリには最初に何かしらのファイルをコミットしておきます。READMEファイルを作りましょう。

$ echo "# 日報作成ツール" > README.md
$ git add README.md
$ git commit -m "Initial commit"

各コマンドの意味：

コマンド何をしているか
echo "..." > README.md「日報作成ツール」というテキストでREADME.mdファイルを作成
git add README.mdファイルを「次の記録に含める」リストに追加（ステージング）
git commit -m "..."変更をメッセージ付きで記録する（コミット）

2-5. GitHubにプッシュする

ローカルのコミットをGitHubに送信します。

$ git branch -M main
$ git push -u origin main
ブラウザで https://github.com/あなたのユーザー名/daily-report-tool を開くと、README.mdが表示されるはずです。

2-6. VS Codeでプロジェクトを開く

ターミナルで以下を実行すると、VS Codeでプロジェクトが開きます。

$ code .

左側のファイルエクスプローラにREADME.mdだけが表示されているはずです。ここにこれからファイルが増えていきます。

STEP 3

CLAUDE.md を作成する

Claude Codeにプロジェクトの文脈を伝える「指示書」を書く ⏱ 5分

CLAUDE.mdは、Claude Codeがセッション開始時に自動で読み込むファイルです。ここに書いた内容がAIの「前提知識」になります。プロジェクトのルートに置きます。

💡 CLAUDE.mdを先に作る理由

Claude Codeを起動した時点でCLAUDE.mdが存在すると、最初の指示から正しい文脈で作業できます。後から追加しても効果はありますが、最初にあるほうが効率的です。

CLAUDE.md の内容

以下の内容で CLAUDE.md を作成します。VS Codeで新規ファイルを作成するか、ターミナルで作成してください。

CLAUDE.md 
# 日報作成ツール
業務で使える日報作成Webアプリ。GitHub Pagesで静的サイトとしてホスティングする。

# 技術スタック
- HTML / CSS / JavaScript（バニラ、フレームワークなし）
- データ保存: localStorage
- ホスティング: GitHub Pages（静的ファイルのみ）

# 機能要件
- 日報入力フォーム（日付、今日やったこと、明日やること、困っていること、メモ）
- localStorageへの保存・読み込み
- 過去の日報の一覧表示
- 日報テキストのクリップボードコピー（Slack/Teams貼付用）
- 日報の削除

# ファイル構成方針
- index.html — メインページ
- css/style.css — スタイル
- js/app.js — アプリケーションロジック
- ファイルは分割して管理する（HTML/CSS/JSを混在させない）

# コーディング方針
- シンプルで読みやすいコードを優先する
- 外部ライブラリは使わない（CDNも含め）
- モバイル対応（レスポンシブデザイン）
- 日本語UIで統一
- アクセシビリティに配慮する（セマンティックHTML、ARIAラベル）

# デザイン方針
- 業務ツールらしい落ち着いた配色
- 余白を十分に取り、情報密度を抑える
- フォントは sans-serif 系のシステムフォント

ターミナルからファイルを作成する場合（VS Codeで作成してもOK）：

$ touch CLAUDE.md

その後、VS Codeで CLAUDE.md を開いて上の内容をコピー＆ペーストし、保存（⌘+S）してください。

💡 「完璧でなくてよい」

CLAUDE.mdは後からいくらでも更新できます。まずはこの最小限の内容で始めて、Claude Codeを使いながら「こういう指示も必要だったな」と気づいた点を追記していくのが、最も実践的な育て方です。

STEP 4

Claude Code で日報ツールを開発する

AIと対話しながら、日報ツールのコードを生成する ⏱ 20–30分

いよいよClaude Codeを使ってコードを書いていきます。ここが本チュートリアルの核心です。

⚠ 重要：ブランチを切ってから作業する

AIにコードを書かせる前に、必ず新しいブランチ（作業用の分岐）を作ります。何か問題が起きても、ブランチごと捨てれば元の状態に戻せます。これがAIコーディングにおける最も重要な安全習慣です。

4-1. 作業ブランチを作成する

$ git checkout -b feature/daily-report-app
Switched to a new branch 'feature/daily-report-app'
これで、mainブランチとは独立した作業場所ができました。

4-2. Claude Code を起動する

プロジェクトフォルダにいることを確認してから、Claude Codeを起動します。

$ claude

Claude Codeが起動し、CLAUDE.mdの内容を自動的に読み込みます。入力プロンプトが表示されたら準備完了です。

4-3. まず計画を立てる（Plan Mode）

いきなりコードを書かせるのではなく、まず計画を確認します。Shift+Tab を2回押して Plan Mode に切り替えてから、以下のように入力してください。

📝 CLAUDE CODE に入力するプロンプト（計画フェーズ）
日報作成ツールの実装計画を立ててください。

以下の機能を含むWebアプリを作ります：
1. 日報入力フォーム（日付、今日やったこと、明日やること、困っていること、メモ）
2. 入力した日報をlocalStorageに保存
3. 過去の日報を一覧表示（日付の新しい順）
4. 各日報をテキスト形式でクリップボードにコピーする機能
5. 日報の削除機能

ファイル構成は CLAUDE.md に記載の通りです。
実装のステップを3〜5個に分割して、各ステップの完了条件を示してください。
コードはまだ書かないでください。

Claude Codeが実装計画を提案してくれます。内容を確認して、問題なければ次に進みます。計画に対して「ステップ2にバリデーションも含めて」のような修正指示を出すこともできます。

💡 Plan Mode の意味

Plan Modeでは、Claude Codeはファイルの変更ができません。調査・分析・計画の提案のみを行います。「考える」と「実行する」を分けることで、意図しないコード変更を防ぎ、人間が計画を確認してから実装に移れます。

4-4. ステップごとに実装する

計画を確認したら、Shift+Tab を2回押してPlan Modeを解除し、実装を依頼します。

📝 CLAUDE CODE に入力するプロンプト（実装フェーズ）
それでは、先ほどの計画に沿って実装を始めてください。

まずステップ1から始めてください。
各ステップが完了したら、動作確認の方法を教えてください。
一度にすべてを作るのではなく、ステップごとに確認しながら進めます。

Claude Codeがファイルを作成し始めます。ファイルの作成や編集の許可を求められたら、内容を確認して承認してください。

💡 許可のプロンプトについて

Claude Codeはファイルを作成・編集する前に許可を求めてきます。表示されたファイル名と変更内容を確認してから y（yes）で承認してください。よくわからない変更であれば n（no）で拒否できます。

4-5. 動作確認する

各ステップの実装後に、ブラウザで動作を確認します。静的なHTML/CSS/JSのプロジェクトなので、index.htmlをブラウザで直接開けます。

$ open index.html
start index.html で既定のブラウザが開きます。あるいは、VS Codeの拡張機能「Live Server」をインストールすると、ファイルを保存するたびにブラウザが自動更新される環境が作れます。

動作確認のポイント：

  • ページが正常に表示されるか レイアウト崩れ、文字化け、真っ白なページなどがないか
  • フォームに入力して保存できるか 各項目に文字を入れて保存ボタンを押し、エラーが出ないか
  • 保存した日報が一覧に表示されるか ページをリロードしても一覧が保持されるか（localStorageの確認）
  • コピー機能が動作するか コピーボタンを押して、テキストエディタに貼り付けてみる

4-6. 問題があれば Claude Code に修正を依頼する

動作確認で問題が見つかった場合、Claude Codeにそのまま伝えれば修正してくれます。

📝 修正依頼の例
日報を保存した後、フォームの入力内容がクリアされません。
保存成功後にフォームをリセットしてください。

あるいは：

📝 修正依頼の例
スマートフォンの画面幅で見ると、フォームがはみ出しています。
モバイル対応のレスポンシブデザインを修正してください。
💡 「何が起きているか」を具体的に伝える

「動きません」より「保存ボタンを押すとコンソールに "TypeError: Cannot read property 'value' of null" というエラーが出ます」のほうが、AIは正確に対処できます。もしエラーメッセージが出ていたら、そのままコピーして貼り付けてください。

4-7. 実装が完了したら

すべての機能が動作することを確認したら、Claude Codeのセッションをクリアします。

> /clear

これでコンテキストがリセットされ、次のタスクに新鮮な状態で臨めます。CLAUDE.mdの内容は保持されます。

完成時のファイル構成（参考）

Claude Codeの出力によって多少異なりますが、おおむね以下のような構成になるはずです。

daily-report-tool/
├── CLAUDE.md ← AIへの指示書
├── README.md ← プロジェクト説明
├── index.html ← メインページ
├── css/
│ └── style.css ← スタイル
└── js/
    └── app.js ← アプリロジック
STEP 5

レビューしてコミットする

AIが書いたコードを確認し、Gitに記録してGitHubにプッシュする ⏱ 5–10分

5-1. VS Code で変更を確認する

VS Codeの左サイドバーにあるソース管理アイコン（分岐のような形のアイコン）をクリックすると、変更されたファイルの一覧が表示されます。

各ファイルをクリックすると、変更前後の差分（Diff）が色付きで表示されます。緑色が追加された行、赤色が削除された行です。

💡 レビューのポイント

すべてのコードを完全に理解する必要はありませんが、以下の点は確認しておくと安心です：
・意図しないファイルが変更されていないか
・HTMLの構造が読み取れる程度に整理されているか
・明らかに不要なコード（テスト用のconsole.logなど）が残っていないか

5-2. ターミナルでも確認する

ターミナルからも変更の概要を確認できます。

$ git status

変更されたファイルの一覧が表示されます。

$ git diff

各ファイルの変更内容が詳細に表示されます（q キーで終了）。

5-3. 変更をコミットする

確認が済んだら、すべての変更をステージング（コミット対象に含める）してコミットします。

$ git add .
$ git commit -m "feat: 日報作成ツールの初期実装"

git add . の .（ドット）は「現在のフォルダ内のすべての変更」を意味します。

5-4. GitHubにプッシュする

$ git push -u origin feature/daily-report-app
GitHubにブランチがプッシュされます。この時点で、GitHubのリポジトリページにアクセスすると、新しいブランチが作成されていることが確認できます。

5-5. mainブランチにマージする

作業ブランチの内容をmainブランチに統合します。

$ git checkout main
$ git merge feature/daily-report-app
$ git push origin main

各コマンドの意味：

コマンド何をしているか
git checkout mainmainブランチに戻る
git merge feature/...作業ブランチの変更をmainに統合する
git push origin main統合結果をGitHubに送信する
💡 プルリクエスト（PR）を使う方法もある

実務やチーム開発では、直接マージするのではなく「プルリクエスト」を作成してレビューしてからマージするのが一般的です。今回は学習のためにシンプルな直接マージを採用していますが、慣れてきたら gh pr create でPRを作成するワークフローも試してみてください。

STEP 6

GitHub Pages で公開する

GitHubの無料ホスティング機能で、日報ツールをインターネットに公開する ⏱ 5分

GitHub Pagesは、GitHubリポジトリの内容をそのままWebサイトとして公開できる無料のホスティングサービスです。静的ファイル（HTML/CSS/JS）をそのまま配信できるため、今回の日報ツールに最適です。

6-1. GitHub Pages を有効化する

gh CLIを使って設定します：

$ gh api repos/{owner}/{repo}/pages -X POST -f "build_type=workflow" -f "source[branch]=main" -f "source[path]=/" 2>/dev/null || echo "Pages may already be enabled or use the web UI below"
💡 ブラウザから設定する方法（より確実）

gh CLIでうまくいかない場合は、ブラウザからの設定が確実です：
1. https://github.com/あなたのユーザー名/daily-report-tool/settings/pages を開く
2. 「Source」で Deploy from a branch を選択
3. Branch を main、フォルダを / (root) に設定
4. 「Save」をクリック

6-2. 公開を確認する

設定後、1〜2分ほど待つとサイトが公開されます。以下のURLでアクセスできます：

https://あなたのユーザー名.github.io/daily-report-tool/

公開状態はGitHubリポジトリのActionsタブで確認できます：

$ gh run list --limit 3
STATUS TITLE BRANCH
✓ pages build and deployment main
ステータスが ✓（チェックマーク）になっていれば、デプロイ完了です。
ページが表示されない場合：
・URLの大文字小文字を確認してください（GitHubのユーザー名は小文字で入力）
・Actionsタブでビルドがまだ実行中かもしれません。数分待ってみてください
・index.html がリポジトリのルート（最上位）に存在するか確認してください

6-3. 動作確認

公開されたURLにアクセスして、以下を確認します：

  • ページが正常に表示される CSSが適用されて、レイアウトが崩れていないか
  • 日報の入力と保存ができる フォームに入力して保存し、ページをリロードしても表示されるか
  • スマートフォンでも使える スマホのブラウザでURLを開くか、PCブラウザの開発者ツールでモバイル表示を確認
  • コピー機能が動作する コピーした内容をSlackやメモ帳に貼り付けて確認
